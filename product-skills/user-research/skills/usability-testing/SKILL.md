# Usability Testing

## Purpose
Discover where users struggle, succeed, and get lost in a product or prototype — through observation of real behavior, not self-report.

## Moderated vs Unmoderated

| | Moderated | Unmoderated |
|---|---|---|
| Format | Live session with researcher | Self-paced, recorded |
| Best for | Complex tasks, nuanced insight | Quick tests, scale, remote |
| Sample size | 5-8 per round | 20-50 |
| Tools | Zoom + screen share, UserTesting | Maze, Lyssna, UserTesting |
| Time to insight | Slower | Faster |

## The 5-User Rule
Jakob Nielsen: 5 users find ~85% of usability issues. Do multiple rounds of 5 with iteration between rounds rather than one large study.

## Task Design

Good tasks are:
- **Realistic**: Based on actual user goals, not feature demonstrations
- **Scenario-based**: "Imagine you just received an invoice for $500. Find the payment status."
- **Not leading**: Don't mention the feature name or location
- **Completable**: Users can succeed or fail clearly

Bad task: "Use the search feature to find the Johnson invoice"
Good task: "You need to check whether you've been paid for the Johnson project. What would you do?"

## Usability Test Protocol

### Before
1. Define 3-5 key tasks
2. Set success criteria per task
3. Prepare think-aloud instructions
4. Test your own test (pilot with a colleague)

### During
- "Please think out loud as you work through this"
- Never help — say "What would you do next if I weren't here?"
- Note: hesitations, errors, backtracking, emotional reactions
- Time on task and success/failure

### After Each Session
- Note top 3 observations immediately (memory fades fast)
- Note direct quotes

## Metrics to Track

| Metric | Definition | Why It Matters |
|---|---|---|
| Task Success Rate | % who complete task correctly | Core usability |
| Time on Task | Time to complete | Efficiency |
| Error Rate | # of errors per task | Learnability |
| Satisfaction (SEQ) | 7-point post-task rating | Subjective ease |
| Think-aloud observations | Confusion points, workarounds | Design input |

**Single Ease Question (SEQ)**: "Overall, how would you rate the difficulty of this task?" (1=Very Difficult, 7=Very Easy). Target >5.5 SEQ.

## Output Format
- Test plan
- Task scripts
- Findings report (by task: success rate, time, key observations, top issues)
- Prioritized redesign recommendations
