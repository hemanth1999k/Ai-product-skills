# Qualitative Research Frameworks

## Purpose
Apply the right qualitative method to the right research question — each method reveals different types of truth.

## Methods Reference

### 1. Jobs-to-Be-Done (JTBD) Interviews
**Best for**: Understanding why customers hire/fire products; uncovering motivation
**Format**: 45-60 min; focus on the purchase/switching moment
**Key questions**:
- Walk me through the moment you decided to try [product/solution]
- What was happening in your life that made you look for something?
- What did you try before? Why wasn't that enough?
- What almost made you NOT choose this?
- When do you feel most/least successful using it?

**Output**: Job statements ("When [situation], I want to [motivation], so I can [outcome]")

### 2. Contextual Inquiry
**Best for**: Understanding how people actually work in context (not what they SAY they do)
**Format**: Observe + interview while user does real work; 60-90 min
**Key behaviors**:
- Watch, don't guide
- Ask "why are you doing that?" in the moment
- Note workarounds, artifacts, environmental factors
- "Think aloud" protocol

**Output**: Workflow documentation, workaround inventory, environmental insights

### 3. Diary Study
**Best for**: Understanding behavior over time; episodic/intermittent behaviors
**Format**: Participants log entries over days/weeks at trigger moments
**Design tips**:
- Provide clear logging prompts
- Use low-friction logging (voice note, text, photo)
- Brief check-in calls mid-study to course correct
- 5-15 participants; 5-21 days

**Output**: Behavioral timeline, longitudinal patterns, context snapshots

### 4. Ethnographic Research
**Best for**: Deep cultural/behavioral understanding; novel domains
**Format**: Researcher immerses in user's environment over days/weeks
**Output**: Rich behavioral insights, cultural patterns, environmental context

### 5. Card Sorting
**Best for**: Information architecture, navigation, mental model mapping
**Open sort**: Users group items into categories they create
**Closed sort**: Users sort items into predefined categories
**Tools**: Optimal Workshop, Maze
**Sample size**: 15-20 per user type

### 6. Tree Testing
**Best for**: Testing navigation structure without visual design distraction
**Format**: Users find items in a text-only version of site structure
**Metrics**: Success rate, time to find, directness

## Choosing the Right Method

| Research Question | Best Method |
|---|---|
| Why do users switch to/from us? | JTBD interview |
| How do users actually use the product? | Contextual inquiry |
| How does behavior change over time? | Diary study |
| Can users find what they need? | Tree testing |
| How do users mentally organize information? | Card sorting |
| What's it like to be our user day-to-day? | Ethnography |

## Output Format
- Method recommendation with rationale
- Study design for chosen method
- Recruitment criteria
- Analysis plan
